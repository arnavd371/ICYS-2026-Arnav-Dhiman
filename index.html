Saddle Escape Efficiency: A Novel Metric to Benchmark Learning Rates in Non-Convex Optimization

Arnav Dhiman, Independent Researcher

March 7, 2026

Introducing SEE, a principled metric for evaluating how efficiently adaptive learning rate frameworks escape saddle points in non-convex optimization.

Read Paper Download PDF

Abstract

Gradient descent is a fundamental optimization algorithm widely used in machine learning and numerical analysis. When it comes to non-convex functions, the choice of a learning rate framework in order to escape saddle points becomes extremely important. Although many studies focus on improving the gradient descent algorithm, this study focuses on how to navigate among current learning rate frameworks in order to choose the best one by introducing a novel metric: Saddle Escape Efficiency (SEE). Existing optimization studies primarily evaluate convergence speed or final loss, whereas SEE directly measures an optimizer's ability to escape saddle regions, a behavior not captured by standard metrics. This study also presents an experimental and computational analysis of three benchmark non-convex functions based on this new metric. We investigate the impact of SEE in order to better and more efficiently choose a learning rate framework which escapes saddle points most efficiently.

The Challenge of Saddle Points

Gradient descent is the backbone of modern machine learning, powering algorithms from linear regression to deep neural networks. One of its key parameters is the learning rate, which controls the step size at each iteration. If the learning rate is too high, the algorithm overshoots the minimum. If it is too low, convergence becomes very slow.

In non-convex optimization, the landscape contains multiple local minima and saddle points. A saddle point is a critical point where the gradient is zero, but the point is neither a local minimum nor a local maximum. These points cause gradient-based algorithms to stagnate or slow down significantly.

Adaptive learning rate methods such as AdaGrad, RMSProp, and Adam are widely used to handle these landscapes. However, there is no dedicated metric to quantitatively evaluate and compare their ability to escape saddle points. Choosing the right optimizer still relies largely on the intuition of the programmer, which becomes increasingly inefficient as functions grow in complexity and dimension.

At a saddle point, the function curves upward in one direction and downward in another. The gradient is zero, but it is not a minimum or maximum.

Saddle Escape Efficiency (SEE)

SEE = Pesc / τavg

  • Pesc is the probability that the optimizer escapes the saddle region within a fixed budget of Tmax = 100 iterations, estimated over N = 50 independent trials. Escape is recorded when the Euclidean norm ||x|| > 2.0.
  • τavg is the average number of steps taken to escape, conditioned on successful escape.

Interpretation: A high SEE value means the optimizer escapes saddle regions quickly and reliably. A low SEE means the optimizer either gets trapped or escapes too slowly. SEE is not a static property of an optimizer. It varies based on the local geometry of the loss landscape.

Initialization: Each trial starts from a small random perturbation around the origin, x0 drawn from Normal(0, 0.01). Learning rates tested: {0.001, 0.01, 0.05, 0.1, 0.2, 0.5}

Benchmark Functions

Himmelblau Function

f(x,y) = (x^2 + y - 11)^2 + (x + y^2 - 7)^2

Used to test escape from a landscape with multiple minima and saddle regions.

Rosenbrock Function

f(x,y) = (1 - x)^2 + 100(y - x^2)^2

A narrow curved valley that is notoriously difficult for gradient-based methods.

Ackley Function

f(x) = -20 * exp(-0.2 * sqrt(mean(xi^2))) - exp(mean(cos(2*pi*xi))) + 20 + e

A highly multimodal function with many local minima and a nearly flat saddle region near the origin.

Findings

RMSProp Leads

RMSProp achieved the highest SEE scores across all three benchmark functions. Its adaptive scaling allows for an immediate increase in step size along flat dimensions without the momentum lag that hindered Adam's performance. At a base learning rate of 0.2 or above, RMSProp reached maximum SEE (1.0) on all three functions.

Adam Underperforms

Despite its popularity, Adam showed poor saddle escape efficiency across all tested functions and learning rates. On the Rosenbrock and Ackley functions, Adam achieved SEE = 0 at every learning rate tested. This is attributed to its momentum accumulation, which dampens the directional sensitivity needed to exit flat saddle regions quickly.

No Universal Optimizer

The optimal learning rate framework varies depending on the geometry of the non-convex landscape. SEE is not a static property of an optimizer. It changes based on the function's local curvature, starting point, and escape threshold. This confirms that SEE provides a geometry-aware criterion for selecting optimizers rather than a one-size-fits-all answer.
